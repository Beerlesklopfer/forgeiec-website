## إدارة هرمية لنظام الحافلات

ينظم ForgeIEC الاتصالات الصناعية في هيكل شرائح
متوافق مع CoDeSys:

```
أنظمة الحافلات
+-- Modbus TCP: القاعة 1 (eth0) [نشط]
|   +-- 192.168.1.100 -- وحدة الحرارة (Slave 1)
|   |   +-- الحرارة : INT (%IW0)
|   |   +-- القيمة_المرجعية : INT (%QW10)
|   +-- 192.168.1.101 -- المضخة (Slave 2)
+-- Modbus RTU: المختبر (/dev/ttyUSB0)
+-- غير مخصص (مجموعة الماسح)
    +-- 192.168.2.55 -- غير معروف
```

## البروتوكولات المدعومة

| البروتوكول | الوسط | مجال الاستخدام |
|-----------|-------|----------------|
| **Modbus TCP** | إيثرنت | أتمتة المباني، تقنية العمليات |
| **Modbus RTU** | RS-485 (تسلسلي) | أجهزة الاستشعار، الأجهزة الميدانية البسيطة |
| **EtherCAT** | إيثرنت (الوقت الفعلي) | التحكم في الحركة، المدخلات/المخرجات السريعة |
| **Profibus DP** | تسلسلي (حافلة ميدانية) | أتمتة التصنيع |

## تعيين العناوين التلقائي

يتم تعيين عناوين IEC (`%IX`، `%QW`، `%MD` إلخ) عالمياً وبدون تعارض.
يتم مراعاة العناوين الحالية في قوائم المتغيرات العامة.

## اكتشاف الأجهزة

يكتشف الماسح الشبكي المدمج الأجهزة المتوافقة مع Modbus تلقائياً.
يمكن تخصيص الأجهزة المكتشفة مباشرة لشريحة.

## تتبع التغييرات

يتم عرض التغييرات في متغيرات الحافلة في حوار فروقات واضح
قبل نقلها إلى نظام التشغيل. يحتفظ المستخدم بالسيطرة الكاملة.
