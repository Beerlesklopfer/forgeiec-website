## السندان: قلب كل مسبك

في كل مسبك، السندان هو القطعة المركزية — حيث يُشكّل المعدن ويُقسّى
ويُصقل. **Anvil Technology\u00ae** هو الطبقة الوسيطة بين نظام تشغيل PLC وجسور ناقل
الحقل. هنا تُصاغ بيانات العملية الخاصة بكم: تُستقبل وتُحوّل وتُوزّع
إلى المستقبلين المناسبين.

Anvil يستخدم داخلياً طبقة نقل ذاكرة مشتركة مملوكة بدون نسخ
للاتصال بين العمليات. بدون تسلسل، بدون نسخ، بدون تنازلات.

## الهندسة المعمارية

```
┌──────────────┐         ┌────────────┐         ┌──────────────────┐
│              │         │            │         │                  │
│ برنامج PLC  │◄───────►│  forgeiecd  │◄───────►│  جسر Modbus      │──► أجهزة حقلية
│  (كود IEC)   │  gRPC   │  (خادم)    │  Anvil  │  جسر EtherCAT    │──► محركات
│              │         │            │ Anvil   │  جسر Profibus     │──► حساسات
└──────────────┘         └────────────┘         │  جسر OPC-UA      │──► SCADA
                                                └──────────────────┘

                         ◄── Anvil ──►
                         IPC بدون نسخ
                         ذاكرة مشتركة
```

يتم تبادل البيانات بين `forgeiecd` وجسور البروتوكول عبر **Anvil Technology\u00ae** —
قناة IPC عالية الأداء مبنية على ذاكرة مشتركة بدون نسخ.
كل قطاع يحصل على قناة اتصال خاصة به.

## لماذا Anvil Technology\u00ae؟

### زمن استجابة بالميكروثانية

آليات IPC التقليدية (الأنابيب، المقابس، طوابير الرسائل) تنسخ البيانات
بين العمليات. Anvil يلغي كل عملية نسخ. البيانات موجودة في الذاكرة
المشتركة — المستقبل يقرأ مباشرة.

| الطريقة | زمن الاستجابة النموذجي | عمليات النسخ |
|---------|----------------------|-------------|
| مقبس TCP | 50–200 ميكروثانية | 2–4 |
| مقبس Unix | 10–50 ميكروثانية | 2 |
| **Anvil Technology\u00ae** | **< 1 ميكروثانية** | **0** |

### جودة صناعية

- سلوك حتمي — بدون تخصيص ذاكرة ديناميكي في المسار الحرج
- خوارزميات بدون قفل — بدون حظر، بدون طريق مسدود
- نموذج نشر/اشتراك — اقتران فضفاض بين المنتج والمستهلك
- إدارة دورة حياة تلقائية — الجسور تُراقب وتُعاد تشغيلها عند الانهيار

### PUBLISH/SUBSCRIBE في برنامج IEC

```iec
VAR_GLOBAL PUBLISH 'Motors'
    K1_Mains    AT %QX0.0 : BOOL;
    K1_Speed    AT %QW10  : INT;
END_VAR

VAR_GLOBAL SUBSCRIBE 'Sensors'
    Temperature AT %IW0   : INT;
    Pressure    AT %IW2   : INT;
END_VAR
```

## البروتوكولات المدعومة

| البروتوكول | الجسر | الحالة |
|-----------|-------|--------|
| **Modbus TCP** | `forgeiec-modbustcp` | متوفر |
| **Modbus RTU** | `forgeiec-modbusrtu` | متوفر |
| **EtherCAT** | `forgeiec-ethercat` | قيد التطوير |
| **Profibus DP** | `forgeiec-profibus` | قيد التطوير |
| **OPC-UA** | `forgeiec-opcua` | مخطط |

كل جسر يعمل كعملية مستقلة. `forgeiecd` يبدأ ويراقب ويعيد تشغيل
الجسور تلقائياً.
