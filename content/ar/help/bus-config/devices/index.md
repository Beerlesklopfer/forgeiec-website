## نظرة عامة

**جهاز الناقل** هو **جهاز واحد داخل شريحة** — عادةً عبد Modbus TCP
(كتلة I/O، محرك)، عبد EtherCAT (محور سيرفو، قارن I/O)، عبد
Profibus DP أو محول EtherNet-IP. لكل جهاز، يحافظ الجسر المسؤول على
اتصال منطقي واحد، يستقصي السجلات المضبوطة وينشر البيانات عبر
مجموعة Anvil IPC إلى وقت تشغيل PLC.

يمكن أن يكون الجهاز **معيارياً**: قارن ناقل (الفتحة 0) يحمل 1..N
وحدة I/O في الفتحات 1..N. الأجهزة المدمجة بدون فتحات توسع تكون قائمة
`modules` فيها فارغة — تعيش المتغيرات حينئذ مباشرة على الفتحة 0.

## حقول الجهاز

تعريف البنية يقع في `editor/include/model/FBusSegmentConfig.h`
(بجوار الشريحة). يُحفظ الجهاز في مشروع `.forge` كـ
`<fi:device>` ضمن `<fi:segment>` (انظر [تكوين الناقل](../)).

### الهوية + العنونة

| الحقل | النوع | المعنى |
|---|---|---|
| `deviceId` | UUID | المفتاح الأساسي المستقر — يُولَّد تلقائياً عند الإنشاء. يصمد أمام إعادة تسمية اسم المضيف وتغيير IP، مما يحافظ على استقرار جميع روابط المتغيرات. |
| `hostname` | string | تسمية مرئية للمستخدم (`"Maibeere"`، `"Stachelbeere"`). آمنة لـ DHCP، لكنها صراحةً **ليست** المفتاح الأساسي. |
| `ipAddress` | string (IP) | عنوان IP (Modbus TCP / EtherNet-IP). فارغ للأجهزة بدون IP (عبيد EtherCAT يعرّفون أنفسهم عبر موضع الناقل). |
| `port` | int | منفذ TCP. الافتراضي `502` (Modbus TCP). |
| `slaveId` | int | معرّف عبد Modbus (1..247). عادةً `1` عبر TCP. |
| `anvilGroup` | string | مجموعة Anvil IPC للنقل بدون نسخ بين الجسر ووقت تشغيل PLC. الاصطلاح: نفس اسم `hostname`. |
| `catalogRef` | string | مرجع اختياري إلى مدخل كتالوج FDD (`"WAGO-750-352"`) يصف الجهاز. |
| `description` | string | وصف نص حر (`"Bewaesserungsventil Sued"`). |

### الوحدات (الفتحات)

| الحقل | النوع | المعنى |
|---|---|---|
| `modules` | قائمة `FBusModuleConfig` | وحدات I/O للجهاز. الفتحة 0 = قارن / جهاز مدمج، الفتحات 1..N = وحدات التوسع. لكل وحدة: `slotIndex`، `catalogRef`، `name`، `baseAddress`، `settings`. |

### تجاوزات لكل جهاز

تتجاوز هذه الحقول — لهذا الجهاز فقط — القيم المقابلة للشريحة. `0`
أو سلسلة فارغة تعني *الوراثة من الشريحة*. في لوحة الخصائص تقع
ضمن كتلة *تجاوزات متقدمة*، مطوية عادة.

| الحقل | النوع | المعنى |
|---|---|---|
| `mac` | string `AA:BB:CC:DD:EE:FF` | عنوان MAC لـ ARP الثابت / فحص الهوية. يحمي من سرقة IP على أجهزة DHCP. |
| `endianness` | enum | ترتيب الكلمات/البايتات للقيم متعددة السجلات: `"ABCD"` (big-endian، افتراضي IEC)، `"DCBA"` (تبديل الكلمات)، `"BADC"` (تبديل البايتات)، `"CDAB"` (تبديل البايتات + الكلمات). فارغ = الوراثة من الشريحة. |
| `timeoutOverrideMs` | int (ms) | مهلة لكل جهاز. `0` = استخدام مهلة الشريحة. |
| `retryCount` | int | عدد إعادة المحاولات لكل طلب. `0` = افتراضي الشريحة. |
| `connectionMode` | enum | `"always"` (الإبقاء على TCP مفتوحاً بين الدورات) أو `"on_demand"` (إعادة الاتصال لكل معاملة). فارغ = افتراضي الشريحة / الجسر. |
| `gatewayOverride` | string (IP) | بوابة لكل جهاز عندما يكون الجهاز في شبكة فرعية مختلفة عن بطاقة شبكة الربط. |

### الإعدادات الخاصة بالجهاز

خريطة `settings` (مفتاح/قيمة) تحمل قيماً منطقية فقط لهذا الجهاز
أو نوع جهازه — مثل عتبة محرك أو رمز وظيفة مفضل.

## سير التحرير

| الإجراء | التأثير |
|---|---|
| **النقر مرة واحدة** على عقدة جهاز | يعرض `FPropertiesPanel` كافة الحقول كمحررات داخلية — كتلة عام (اسم المضيف، IP، المنفذ، معرّف العبد، مجموعة Anvil)، كتلة التجاوز (MAC، المهلة، إعادة المحاولات، endianness، وضع الاتصال، تجاوز البوابة، الوصف) وجدول الحالة. |
| **النقر المزدوج** على عقدة جهاز | يفتح `FBusDeviceDialog` المشروط بنفس مجموعة الحقول. في وضع التحرير يكون زر "استيراد من الكتالوج" مقفلاً حتى لا يقوم استيراد FDD لاحق بالكتابة فوق روابط متغيرات I/O الموجودة بصمت. |

## متغيرات الحالة (للقراءة فقط)

في وقت التشغيل ينشر كل جهاز بنية حالة يرسلها البرنامج الخفي عبر
دفق حالة gRPC. تُعرض هذه القيم في لوحة الخصائص كـ **جدول للقراءة فقط**
وهي **غير قابلة للتحرير** من واجهة المستخدم — يكتبها الجسر. من كود ST
لا تزال قابلة للعنونة كمسارات مؤهلة تحت `anvil.<seg>.<dev>.Status.*`:

| متغير الحالة | النوع | المعنى |
|---|---|---|
| `xOnline` | `BOOL` | الجهاز قابل للوصول حالياً (آخر طلب أُجيب). |
| `eState` | `INT` | تعداد الحالة: 0=offline، 1=connecting، 2=online، 3=error. |
| `wErrorCount` | `WORD` | عدّاد الطلبات الفاشلة منذ بدء الجسر. |
| `sLastErrorMsg` | `STRING` | آخر رسالة خطأ (مهلة، استثناء Modbus، ...). |

```iec
IF anvil.Halle1.Maibeere.Status.xOnline AND
   anvil.Halle1.Maibeere.Status.wErrorCount < 10 THEN
    bSensor_OK := TRUE;
END_IF;
```

## مثال: قارن ناقل WAGO 750 بفتحتين

قارن ناقل Modbus TCP 750-352 مع وحدة 8-DI واحدة (750-430) في الفتحة 1
ووحدة 8-DO واحدة (750-530) في الفتحة 2:

```toml
[[bus_segments.devices]]
device_id    = "0e5d5537-e328-44e6-8214-78d529b18ebd"
hostname     = "Maibeere"
ip_address   = "192.168.24.25"
port         = 502
slave_id     = 1
anvil_group  = "Maibeere"
catalog_ref  = "WAGO-750-352"
description  = "Bus coupler hall 1, row A"

[[bus_segments.devices.modules]]
slot_index   = 0
catalog_ref  = "WAGO-750-352"
name         = "Coupler"
base_address = 0

[[bus_segments.devices.modules]]
slot_index   = 1
catalog_ref  = "WAGO-750-430"
name         = "8 DI Slot 1"
base_address = 0     # Coil 0..7

[[bus_segments.devices.modules]]
slot_index   = 2
catalog_ref  = "WAGO-750-530"
name         = "8 DO Slot 2"
base_address = 0     # Discrete Output 0..7
```

تظهر المدخلات الثمانية في مجمّع العناوين كـ `%IX0.0..%IX0.7` بـ
`deviceId="0e5d5537-..."`، `moduleSlot=1` و `modbusAddress=0..7`.
المخرجات الثمانية كذلك بـ `moduleSlot=2`.
