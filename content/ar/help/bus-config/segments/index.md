## نظرة عامة

**شريحة الناقل** تصف **شبكة فيزيائية واحدة على واجهة واحدة من هدف PLC**
— عادةً منفذ Ethernet (`eth0`، `enp3s0`) لـ Modbus TCP / EtherCAT /
EtherNet-IP، أو منفذ تسلسلي (`/dev/ttyUSB0`) لـ Modbus RTU / Profibus DP.
لكل شريحة، يقوم البرنامج الخفي `anvild` بإطلاق **عملية جسر واحدة بالضبط**
(`tongs-modbustcp`، `tongs-ethercat`، ...) تتعامل مع حركة المرور إلى
جميع الأجهزة في تلك الشريحة.

يمكن لمشروع أن يحوي أي عدد من الشرائح — لكل منها بروتوكولها الخاص،
وواجهتها الخاصة، وإيقاع الاستقصاء الخاص بها. على سبيل المثال يمكن
لمتحكم محور EtherCAT سريع (`eth1`، 1 ms) ومستقصي مستشعر Modbus TCP بطيء
(`eth0`، 100 ms) أن يعملا جنباً إلى جنب في نفس المشروع.

## حقول الشريحة

تعريف البنية يقع في `editor/include/model/FBusSegmentConfig.h`.
تُحفظ الشريحة في مشروع `.forge` كـ `<fi:segment>` ضمن
`<fi:busConfig>` (انظر [تكوين الناقل](../)).

### الهوية + البروتوكول

| الحقل | النوع | المعنى |
|---|---|---|
| `segmentId` | UUID | المفتاح الأساسي المستقر — يُولَّد تلقائياً عند الإنشاء، غير قابل للتعديل. يصمد أمام إعادة التسمية وتغيير البروتوكول وتغيير IP. |
| `protocol` | enum | `modbustcp` / `modbusrtu` / `ethercat` / `profibus` / `ethernetip`. يحدد أي برنامج خفي للجسر يُشغّل. |
| `name` | string | تسمية المستخدم (مثل `"Fieldbus Hall 1"`). نموذج حر، يُعرض في الشجرة وفي السجلات. |
| `enabled` | bool | مفتاح تشغيل/إيقاف. `false` = الجسر لا يُشغَّل، الأجهزة تبقى دون اتصال. الافتراضي: `true`. |

### الواجهة + التوجيه

| الحقل | النوع | المعنى |
|---|---|---|
| `interface` | string | واجهة الشبكة (`eth0`، `enp3s0`، `/dev/ttyUSB0`). يمررها الجسر إلى واجهة برمجة المقبس / التسلسلية. |
| `bindAddress` | string (IP/CIDR) | عنوان IP المصدر للاتصالات TCP الصادرة، مثل `192.168.24.100/24`. فارغ = نظام التشغيل يختار أول IP للواجهة. |
| `gateway` | string (IP) | البوابة الافتراضية للحزم المغادرة للشبكة الفرعية المحلية. فارغ = لا توجد بوابة. |
| `pollIntervalMs` | int (ms) | فترة استقصاء الجسر. `0` = بأسرع ما يمكن (حلقة مشغولة / زمن حقيقي). نموذجي: `100` لـ Modbus TCP، `0` لـ EtherCAT. |

### إعدادات الشبكة (متقدم)

أُضيفت هذه الحقول في تطوير إعدادات الشبكة وتغطي الحالات التي
لا تكفي فيها افتراضيات نظام التشغيل — عادةً: العديد من اتصالات TCP
المتوازية لكل عبد، جلسات TCP طويلة الأمد عبر NAT، أو شبكات فرعية متعددة
على بطاقة شبكة واحدة.

| الحقل | النوع | المعنى |
|---|---|---|
| `subnetCidr` | string (CIDR) | الشبكة الفرعية المحلية للشريحة، مثل `192.168.24.0/24`. تتيح للجسر التوجيه الصحيح لتجاوزات بوابة كل جهاز عند حمل بطاقة الشبكة عدة شبكات. |
| `sourcePortRange` | string `"min-max"` | تجمع منافذ TCP المصدر للاتصالات الصادرة، مثل `30000-39999`. فارغ = نظام التشغيل يختار من النطاق العابر. مهم عند الحاجة لكثير من الاتصالات المتوازية مع نفس العبد (اتصال واحد لكل منفذ مصدر). |
| `keepAliveIdleSec` | int (s) | ثوانٍ الخمول قبل إرسال أول مسبار keep-alive TCP. `0` = افتراضي نظام التشغيل. |
| `keepAliveIntervalSec` | int (s) | الفاصل بين مسبارات keep-alive. `0` = افتراضي نظام التشغيل. |
| `keepAliveCount` | int | عدد المسبارات الفاشلة قبل إعلان وفاة الاتصال. `0` = افتراضي نظام التشغيل. |
| `maxConnections` | int | الحد الأعلى لتجمع الاتصالات. `0` = غير محدود. مفيد ضد العبيد ذوي حد الاتصال الصارم. |
| `vlanId` | int (1..4094) | علامة VLAN 802.1Q للإطارات الصادرة. `0` = بدون علامة. |

### الإعدادات الخاصة بالبروتوكول

خريطة `settings` (مفتاح/قيمة) تحتفظ بكافة القيم المنطقية لبروتوكول
محدد فقط — مثلاً لـ Modbus TCP: `port`، `timeout_ms`؛ لـ Modbus
RTU: `serial_port`، `baud_rate`، `parity`، `stop_bits`؛ لـ Profibus:
`master_address`. كذلك `log_level` و `log_file` يُحفظان بطريقة محايدة
عن البروتوكول في نفس الخريطة.

## سير التحرير

في لوحة شجرة الناقل كلا المسارين متكافئان — يعملان على نفس
مجموعة الحقول ولهما نفس التأثير الدلالي:

| الإجراء | التأثير |
|---|---|
| **النقر مرة واحدة** على عقدة شريحة | يعرض `FPropertiesPanel` (الإرساء الافتراضي: الجانب الأيمن) كافة الحقول كمحررات داخلية — تُكتب التغييرات في المشروع عند `editingFinished` وتعلّم المشروع كمتسخ. |
| **النقر المزدوج** على عقدة شريحة | يفتح `FSegmentDialog` المشروط بنفس مجموعة الحقول، مجمّعة في *عام* / *Modbus TCP* / *الشبكة المتقدمة* / *التسجيل*. OK يثبّت، Cancel يُلغي. |

## مثال: شريحة Modbus TCP

```toml
[[bus_segments]]
segment_id     = "a3f7c2e1-7c4f-4e1a-9f9c-1a2b3c4d5e6f"
protocol       = "modbustcp"
name           = "Feldbus Halle 1"
enabled        = true
interface      = "eth0"
bind_address   = "192.168.24.100/24"
gateway        = ""
poll_interval  = 100   # ms

[bus_segments.settings]
port           = "502"
timeout_ms     = "2000"
log_level      = "info"
log_file       = "/var/log/forgeiec/halle1.log"
```

تُشغّل هذه الشريحة `tongs-modbustcp` على `eth0` بـ IP المصدر
`192.168.24.100`، تستقصي جميع الأجهزة كل 100 ms، وتقبل حتى 2000 ms
من زمن الاستجابة لكل طلب قبل إصدار خطأ مهلة على دفق الحالة.
