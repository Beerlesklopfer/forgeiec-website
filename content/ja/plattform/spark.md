## Spark -- Zenoh トンネル

Spark は ForgeIEC プラットフォームのエッジからクラウドへのネットワークブリッジ
です。火花が炎を点けるように -- Spark は現場の PLC とクラウドサービス間の
接続を点火します。

---

## 妥協のないエッジからクラウドへの接続

現代の産業設備は、フィールド機器とクラウドサービス間の信頼性の高い接続を必要と
します -- リモートメンテナンス、データ分析、遠隔監視のために。Spark は Zenoh
プロトコルを通じてこの接続を提供します。

### なぜ Zenoh なのか？

Zenoh は制約のある分散環境向けに設計された通信プロトコルです。従来の VPN や
MQTT 接続とは異なり、Zenoh は以下を提供します：

- **ネイティブ NAT トラバーサル** -- 複雑なファイアウォール設定不要
- **効率的な pub/sub プロトコル** -- 低帯域幅消費
- **アダプティブルーティング** -- 最適なネットワークパスの自動選択
- **最小レイテンシ** -- リアルタイムアプリケーション向け設計

---

## ユースケース

### リモートメンテナンス

遠隔地の PLC への安全な接続 -- 診断、プログラム更新、変数読み取り。現地への
移動は不要です。

### クラウドデータ収集

プロセスデータをクラウドプラットフォーム（AWS、Azure、プライベートインフラ）に
転送 -- 分析、機械学習、予知保全のために。

### マルチサイト監視

複数の設備を単一のポイントから集中監視。リアルタイムデータ、最小レイテンシ。

---

## アーキテクチャ

Spark は PLC 上のデーモンとして動作し、Anvil（ゼロコピー IPC）を通じてランタイム
に接続されます。データはリモート Zenoh ノードに選択的に転送されます。

```
サイト A                         クラウド / 中央サイト
+------------+                   +------------------+
| anvild     |                   | Zenoh ルーター    |
|   |        |                   |   |              |
|   +- Spark |----- Zenoh ---------|  +- サービス   |
|   |  Anvil |   (暗号化)        |     分析         |
+------------+                   +------------------+
```

### 特徴

- エンドツーエンド暗号化（TLS 1.3）
- 転送変数の設定可能なフィルタリング
- ネットワーク切断時の自動再接続
- 低帯域幅リンク向けのデータ圧縮
- モバイルネットワーク対応（4G/5G）

---

## 技術詳細

- **プロトコル**：Zenoh（ゼロオーバーヘッドネットワークプロトコル）
- **トランスポート**：TCP、UDP、WebSocket
- **暗号化**：TLS 1.3
- **プラットフォーム**：x86_64、ARM64、ARMv7（Linux）
- **統合**：Anvil IPC でランタイムに接続、Zenoh でクラウドに接続
