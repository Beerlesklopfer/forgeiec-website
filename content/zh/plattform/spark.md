## Spark -- Zenoh 隧道

Spark 是 ForgeIEC 平台的边缘到云端网络桥接。火花点燃火焰 -- Spark 点燃现场
PLC 与云服务之间的连接。

---

## 无妥协的边缘到云端

现代工业设施需要在现场设备和云服务之间建立可靠的连接 -- 用于远程维护、数据分析
和远程监控。Spark 通过 Zenoh 协议提供这一连接。

### 为什么选择 Zenoh？

Zenoh 是一个为受限和分布式环境设计的通信协议。与传统 VPN 或 MQTT 连接不同，
Zenoh 提供：

- **原生 NAT 穿透** -- 无需复杂的防火墙配置
- **高效的发布/订阅协议** -- 低带宽消耗
- **自适应路由** -- 自动选择最佳网络路径
- **最小延迟** -- 为实时应用设计

---

## 应用场景

### 远程维护

安全连接到远程 PLC 进行诊断、程序更新和变量读取 -- 无需亲临现场。

### 云端数据采集

将过程数据传输到云平台（AWS、Azure、私有基础设施）用于分析、机器学习和
预测性维护。

### 多站点监控

从单一控制点集中监控多个设施，实时数据和最小延迟。

---

## 架构

Spark 作为守护进程在 PLC 上运行，通过 Anvil（零拷贝 IPC）连接到运行时。
数据被选择性地传输到远程 Zenoh 节点。

```
站点 A                           云端 / 中心站点
+------------+                   +------------------+
| anvild     |                   | Zenoh 路由器      |
|   |        |                   |   |              |
|   +- Spark |----- Zenoh ---------|  +- 服务       |
|   |  Anvil |   (加密)          |     数据分析      |
+------------+                   +------------------+
```

### 特性

- 端到端加密（TLS 1.3）
- 可配置的变量传输过滤
- 网络中断时自动重连
- 低带宽链路的数据压缩
- 兼容移动网络（4G/5G）

---

## 技术细节

- **协议**：Zenoh（零开销网络协议）
- **传输**：TCP、UDP、WebSocket
- **加密**：TLS 1.3
- **平台**：x86_64、ARM64、ARMv7（Linux）
- **集成**：Anvil IPC 连接运行时，Zenoh 连接云端
